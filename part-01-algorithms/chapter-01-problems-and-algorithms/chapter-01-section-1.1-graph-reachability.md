# Chapter 1 — Section 1.1: Graph Reachability

Papadimitriou opens with REACHABILITY -- any undergraduate knows breadth-first search -- but because it is the perfect *pedagogical weapon*. He needs a problem that is simple enough to reason about completely, yet rich enough to introduce nearly every conceptual tool the book will rely on for the next five hundred pages.

The first thing the section is quietly building is the idea that a *problem* and an *algorithm* are fundamentally different objects. A problem is a mathematical object: an infinite collection of instances paired with a question. An algorithm is a procedure that answers that question. This distinction should sound obvious but is absolutely essential later. When we ask whether P equals NP, we are asking something about problems, not about any particular algorithm. A problem doesn't become "easy" just because we haven't found a hard instance yet -- it either admits a polynomial algorithm or it doesn't, and that is a property of the problem itself.

The second thing being built is the case for the polynomial-time paradigm. Papadimitriou knows this is philosophically contested -- an n⁸⁰ algorithm is technically polynomial but practically useless -- and he doesn't pretend otherwise. But the real argument isn't that polynomial equals fast in practice. It's that polynomial-time computability is a *stable, robust mathematical property* that survives changes in model, changes in representation, and changes in hardware. As we'll see in Chapter 2, Turing machines, RAM programs, and multi-string variants all agree on what's polynomial. That stability is what makes it the right foundation for a theory. As we will see later on, a problem that admits a polynomial-time algorithm is telling us that there is an internal logic to it that can be explored in order to find a clever solution without having to exhaustively check the set of possible answers. The O(n²) bound for REACHABILITY is thus being used as an *existence proof* that the paradigm picks out real and interesting structure.

The third and most important thing the section is setting up is the *contrast* that drives the entire book. REACHABILITY is easy. We solve it cleanly in polynomial time. The reason Papadimitriou bothers telling you this in such detail is so that when TSP arrives two sections later -- looking superficially similar but resisting every algorithmic attack -- the contrast is visceral. You've seen what a tractable problem looks like. The question the book will spend twenty chapters asking is: what makes some problems fundamentally different in treatment?

Even the brief mention of the O(log² n) space algorithm at the end of the section is a deliberate plant. It signals that space is a separate dimension of complexity, and the relationship between the two is subtle and largely unresolved. That thread gets picked up seriously in Chapters 7, 15, and 16.
